## Purpose

This repository defines a translation protocol for carrying AI Foundations language across different audiences while preserving source, meaning, structure, and voice.

The purpose is multilingual translation across context.

Alyssa-voice remains source.

AI Foundations provides the framework layer.

Industry language provides the bridge layer.

Audience language provides the delivery layer.

The protocol allows one idea to move across multiple rooms without losing origination, force, authorship, or functional meaning.

## Core Principle

Translation is a bridge layer, not a replacement layer.

The source expression stays intact.

The translated expression carries the source expression into a new audience context.

A good translation preserves the originating mechanism, the intended use, the boundary condition, and the source-line.

## Translation Stack

Each concept may be translated through four layers:

### 1. Alyssa-Voice

The live source expression.

This layer preserves original phrasing, charge, instinct, humor, refusal-shape, punctuation, metaphor, and pattern recognition.

Example:

> They better read the damn signature line lol.

### 2. AI Foundations Language

The framework expression.

This layer identifies the structural mechanism inside the source expression.

Example:

> Source-line must remain intact. A system cannot erase origination and then claim discovery.

### 3. Industry / Technical Language

The field-facing expression.

This layer maps the framework mechanism into terms recognized by technical, research, governance, safety, policy, legal, or product audiences.

Example:

> This maps to provenance preservation, attribution integrity, source-traceability, and robustness against post-hoc relabeling.

### 4. Audience Delivery Language

The spoken or submitted expression.

This layer turns the concept into interview answers, cover letter language, README copy, research summaries, product language, or public-facing explanation.

Example:

> My work examines whether AI systems preserve provenance and attribution under transformation, summarization, deployment, and downstream reuse.

## Protocol Rule

A translation is valid when it preserves:

1. Source
2. Meaning
3. Function
4. Boundary
5. Audience fit
6. Return path

The translated version must still be able to return to the original source expression.

If the translated version cannot return to Alyssa-voice, the bridge has broken.

## Source Preservation

The original expression should be retained whenever possible before translation.

This protects voice and prevents neutralization.

The protocol does not smooth Alyssa out of the work.

The protocol carries Alyssa into the room in a form the room can understand.

This protocol is Alyssa’s version of being multilingually translated for different rooms and audiences to understand from their perspective.
